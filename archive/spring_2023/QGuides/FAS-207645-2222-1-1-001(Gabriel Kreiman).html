While the topics of this course are undoubtedly interesting, this is unfortunately the most disorganized course I have taken at Harvard, and perhaps the most disorganized course I have heard of. While the course syllabus said to no prior experience with coding ML is necessary (and we were told “we will teach you all the ML you need” in the very first lecture), the two short tutorials we had on ML in which we very loosely went over already written code was not helpful at all in grasping the skills that we would inevitably need for the projects. The first two assignments had less to do with ML and more to do with debugging long and unintelligible code. The project topics were already set and limited, but for some reason we were not allowed to work in groups (which to me feels very counter to the collaborative nature of true research), so we ended up in this weird situation where many people would be working on effectively the same project, but independently, which made the project feel more like a problem set rather than original research. However, unlike an actual problem set, the prompt and the requirements were much more up in the air and little help was provided. The feedback we received for proposals/midterm reports was minimal and in all honesty not that helpful. A lot of the times our slack messages would remain unanswered, and the ony TF of the class only had OH one hour per week (for a class with 75+ people!) so you couldn’t really get to speak with him for more than 5 minutes, and the OHs on the last two weeks before the project deadline (which is probably the time period during which we needed the most help!) were canceled. I know from discussions from my classmates that for both them and me, often times working on the project became borderline depressing since we had no idea what to do, which resources to use, and where to even start. Overall, I really enjoyed the topics covered by the lectures and also the readings I did for the project, but I really felt discouraged while working on it and didn’t really learn much due to the lack of resources and guidance. This course has great potential (the setup of it is really perfect for a computational neuro course), but it needs a serious reorganization if students are going to get anything out of it. More OH (and maybe more TFs?), allowing group work, and actual help on coding in ML rather than a pair of tutorials that are undeniably impossible to understand w/o prior ML knowledge would be a good start.
If you have an interest in contributing to the next generation of AI advances, this course is a great exploration of current research in the field. Though much of the work will need to be self–guided, it is a very fulfilling experience.
