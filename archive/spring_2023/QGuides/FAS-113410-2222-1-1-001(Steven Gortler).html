This is probably one of the easiest courses in the CS department –– not in the sense that it doesn't require any time, it still does, but it's not very conceptually difficult. You learn a lot, but really the hardest part of this course is installation. The pset materials are old and you will inevitably have setup issues (that you have to grapple with every single pset). To do well in this course, get very good at reading the pset instructions and following them precisely. Certainly the psets are good at highlighting when I'm failing to understand something conceptually, but most of the time, I spend hours debugging something because I missed a small sentence or phrase in the pset telling me to do something different.

Seriously, read the whole thing before even attempting to download the code. There are instructions in the weirdest spots and missing them wastes hours.
Start the problem sets early. The material is not too difficult and doing the assignments is actually quite fun. However, there may be bugs or issues that come up that can hamper your progress, so it's good to give yourself as much time as possible.
Also, the textbook is VERY helpful! Try to get your hands on a copy if you can.
Get started on the Psets during office hours, because some of the code migration can be a nightmare for the first parts of the Psets.
Ok so quick summary: this class is kind of a mess and could be less painful than other CS classes in the end, but it still is very difficult. The lectures in this class are really good so please go! Sometimes there would be only 6 people at lecture. Prof. Gortler is so funny and his pace is really reasonable compared to other CS classes. Conceptually lectures are pretty easy to understand. However, the problem sets, especially in the second half of the semester, are very difficult. The class has a HUGE jump in difficulty after spring break. In the first part of the semester, the psets were shorter, and sometimes we would have a break with no pset. When we got back from spring break, we had 5 straight very difficult psets. Make sure you know C++, definitely at least the level of 61, although the C++ in this class is a lot harder. No exams in this class which is nice. The psets in this class were really hard because your code doesn’t really matter, the graphics image output from your code has to be right. Also note this is a partner pset class which I didn’t realize. I wish I had a partner going into the class, definitely would have made my life a lot easier. So make sure you have a partner going in. I hope the class improves the code setup process by next year because it is very painful just to get the initial code to compile because it’s incorrect and the makefile is wrong, which is really annoying and not fair to students' time. If you are Mac User, I recommend using a makefile, not Xcode. It's so much easier to get the code running with make.
If you are interested at all in how computers render graphics, TAKE THIS CLASS. It is structured and paced so well, and each assignment is very informative and engaging. Shlomo and the TFs also have an incredible understanding of the content and will help at any point of confusion.
I think this is a great CS course to take. I really enjoyed this class and think it was one of the most manageable CS courses I've taken here. The problem sets were actually kind of fun and not super difficult, and it's cool to see how much progress you make with graphics over the semester.
I really enjoyed this class. Shlomo was such a fun and great professor. He was also always willing to answer emails / questions and ready to help! Would definitely recommend this as one of the electives. It is not a low workload class at all – and psets get longer and harder over the semester – but for an "upper level" CS class it is lower workload. It is very interesting and gives a great overview of graphics. It is coding heavy with quite a lot of matrices so if you're not comfortable with that then consider carefully. I can highly recommend this class!!
Start the PSETs early and make sure you come into the class with an attitude to learn. Show up to class. I know you’ll be tempted not too because it’S in the SEC, but it’s worth it because you get to interact with Professor Gortler firsthand and meet your classmates. Also, definitely come to office hours! I’ve learned so much from my TFs there and met cool students there! Most importantly, please follow up and dive into the lecture material after class if you want to excel. Many of these concepts are new and challenging, so you won’t be able to unpack everything in a single 75 minute lecture. Finally – do some computer graphics research outside of class! By this I mean, Google any topics that interest you related to computer graphics and ask professor about it! This will ensure you’re engaged in the course.

From a coding perspective, it is not akin to CS171, it's much more rigorous. You can get through it, but don't go in thinking it'll just be a fun course, it'll take a lot of work. This course is great. The professor is a fun lecturer and the material is very interesting (learning how to craft 3D robots, make them move, etc.). Psets can be difficult at times when you get stuck, but the TFs and office hours will definitely help with that.
This course isn't going to teach you how to make a game or animate a movie. Instead, it teaches you the low level concepts used by game engines and animation software to create the images in the first place. If you find this interesting then I would definitely recommend the course.
