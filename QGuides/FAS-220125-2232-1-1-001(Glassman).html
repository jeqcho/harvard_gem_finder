This class was offered for the first time this year, so I'm sure things can/will change and will be improved, but I overall learned a lot! If you have taken CS50/51 + an ML/NLP class and have some case team experience from consulting/data analytics/product management, you should be ready to tackle this class.
This class will teach you how to use some popular JavaScript frameworks to do a lot of front–end development and redesign interfaces that you don't like (it will also teach you how to articulate *what* exactly you don't like about those interfaces more clearly). It will also give you more experience articulating and illustrating the design aspects of your code rather than just focusing on the nitty gritty of the implementation (but still making that connection between Figma/hand sketches and actual implementations because software ultimately needs to work). You'll also read a lot of informative research papers from ACM CHI about user interface design/some AI–related human–computer interaction topics, and you will have to make comments on Perusall/interact with your classmates' comments on Perusall on a regular basis. In class, you will probably have to do a lot of small–group discussion and participate actively to get a good participation grade. The big assignments are three projects, where you work in pairs to build a web app that fulfills a certain purpose shared with the class.
If the course stays the same as it did this year, I'll say that I enjoyed the following:
– Reading research papers and, to an extent, reading chapters from Daniel Jackson's Software Concepts
– Learning Svelte.js, React.js, and Flask and using them to build projects
– Learning how to articulate design illustrations and arguments more clearly
If you're thinking about this class, I'd warn you about the following:
– There are comprehension quizzes in almost each class (about the readings due before that class), and most of them are correctness–based, so that can be a bit stressful
– You will be forced to interact with peers (both in in–class discussions, which are then sort of "graded" after each class because you fill out a checkout form evaluating your peers' participation that day, and on each project). For neurodivergent folks, this aspect might be overwhelming.
– Attendance is required and graded.
– There is a weekly required Studio (essentially a section), which I didn't find very useful or fulfilling. Nothing against the TFs leading them; the goal/content was just not very helpful.
– The course staff this year did not wear masks in class, so even though some notions of accessibility and social justice are discussed, take them with a grain of salt. I would say this class is one of the more high–risk for exposure to COVID (and god knows what else that will spawn in the next few years because we are letting COVID spread like this) because of the combination of a lack of masks (even when people are visibly sick) and the requirement for attendance. If you need to take this class but have a limited budget for masks, make sure to save your high–quality N95/KN95+ respirators for use in this class because you will need it.

If you feel tired of Harvard CS theoretical–ness – this course is your cure. You work on three projects over the semester that were interesting and exciting and made me remember why I ever liked coding in the first place.

Please don't let any negative reviews sway you from taking this course. This was the first time this course was taught, so Elena was still figuring out the structure and organization, but I have full faith that this class will only improve. Elena feels more like a friend and partner in the class than a professor, and she cares about the students.

Most of our issues with the class were growing pains (this was the first year it was offered). I think this course has potential to be one of the best classes in the CS department, after some iteration. You learn about concepts, programming frameworks (like Svelte and React), and taking an idea, designing it, and finally building it out into a real app (we did this three times this semester). Super great class, don't be afraid to take it! You get a lot of support but also freedom if you're an experienced coder!

If you are hoping to get applied software development practice at Harvard, take this class before you graduate! Compared to more theoretical CS classes at Harvard, CS 178 gives you a great opportunity to gain a stronger skill in practical software engineering, building effective software interfaces, learning how to quickly use software libraries and frameworks, and work with others to build usable systems. At the same time, if you have experience doing software development either from summer internships or other clubs like HCS & Datamatch, this class may not feel too revolutionary, but going through 3 projects throughout the semester with classmates is very good practice nonetheless, and this class's focus on "software concepts", while simple, will only help you build more usable and intuitive software.
